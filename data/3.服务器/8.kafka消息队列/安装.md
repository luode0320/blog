
## 创建目录

```sh
mkdir -p /usr/local/src/kafka
cd /usr/local/src/kafka
```

# 第一次启动

- 为什么要第一次启动, 可能是这个版本的问题, 有一些隐藏文件需要先拿出来
- 否则不能挂载数据目录

```sh
docker run -d \
  --restart=always \
  --name kafka \
  -e KAFKA_CFG_NODE_ID=1 \
  -e KAFKA_CFG_PROCESS_ROLES=controller,broker \
  -e KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://192.168.1.16:9092 \
  -e KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER \
  -e KAFKA_CFG_LISTENERS=PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093 \
  -e KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT \
  -e KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@0.0.0.0:9093 \
  -e KAFKA_KRAFT_CLUSTER_ID=abcdefghijklmnopqrstuv \
  -e KAFKA_CFG_LOG_RETENTION_BYTES=524288000 \
  -e KAFKA_LOG_DIRS=/bitnami/kafka/data \
  -p 9092:9092 \
  -p 9093:9093 \
  luode0320/kafka:latest
```

## 拷贝基础数据

- 其实拷贝的是一些隐藏文件, ls命令是看不到的

```sh
docker cp kafka:/bitnami/kafka/data /usr/local/src/kafka/
#Successfully copied 3.58kB to /usr/local/src/kafka/
```

## 删除源容器

```sh
docker rm -f kafka
```

# 启动

```sh
docker run -d \
  --restart=always \
  --name kafka \
  -e KAFKA_CFG_NODE_ID=1 \
  -e KAFKA_CFG_PROCESS_ROLES=controller,broker \
  -e KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://192.168.1.16:9092 \
  -e KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER \
  -e KAFKA_CFG_LISTENERS=PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093 \
  -e KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT \
  -e KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@0.0.0.0:9093 \
  -e KAFKA_KRAFT_CLUSTER_ID=abcdefghijklmnopqrstuv \
  -e KAFKA_CFG_LOG_RETENTION_BYTES=524288000 \
  -e KAFKA_LOG_DIRS=/bitnami/kafka/data \
  -v /usr/local/src/kafka/data:/bitnami/kafka/data \
  -p 9092:9092 \
  -p 9093:9093 \
  luode0320/kafka:latest
```

# 说明

```sh
- KAFKA_CFG_NODE_ID=1 # 节点的 ID，这个 ID 在 Kafka 集群中必须是唯一的
- KAFKA_CFG_PROCESS_ROLES=controller,broker # 节点既是控制器又是代理，负责集群的控制逻辑又负责消息的存储和转发
- KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://192.168.1.16:9092 # 外网IP的 9092 端口
- KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER # 控制器监听器的名称，这里设置为 CONTROLLER
- KAFKA_CFG_LISTENERS=PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093 # 两个监听器
- KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT # 两个监听器使用明文传输协议
- KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@0.0.0.0:9093 # 选举投票者，使用 0.0.0.0 的 9093 端口
- KAFKA_KRAFT_CLUSTER_ID=abcdefghijklmnopqrstuv # Kafka Kraft 集群的 ID，用于标识属于同一个 Kraft 集群的 Kafka 节点
- KAFKA_CFG_LOG_RETENTION_BYTES=524288000 # 设置日志保留的字节数，这里设置为 500M=524288000 1GB=1073741824 5GB=5368709120
- KAFKA_LOG_DIRS=/bitnami/kafka/data
```
