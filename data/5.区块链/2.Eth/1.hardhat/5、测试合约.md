## 测试合约

为智能合约编写自动化测试至关重要，因为事关用户资金。

为此，我们将使用**Hardhat Network**，这是一个内置的以太坊网络，专门为开发设计，并且是**Hardhat**中的默认区块链网络。

无需进行任何设置即可使用它。 在我们的测试中，我们将使用[ethers.js](https://learnblockchain.cn/ethers_v5/)
与前面构建的合约进行交互，并使用 [Mocha](https://mochajs.org/) 作为测试运行器。

我们现在还没有学习 [ethers.js](https://learnblockchain.cn/ethers_v5/) 与  [Mocha](https://mochajs.org/) , 但是没关系,
我们现在只是搭建一个简单案例模板, 我们只需要先熟悉整个流程即可。

## 编写测试用例

在项目根目录中创建一个名为`test`的新目录，并创建一个名为`test.js`的新文件(默认生成的 `test\Lock.js` 已经删除)

```js
// 导入 Chai 库中的 expect 函数，用于断言测试结果。
const { expect } = require("chai");

// 使用 describe 定义一个测试套件，描述为 "Token contract"。
describe("describe: Token 智能合约", function () {
    // 使用 it 定义一个测试案例，描述为 "部署应将令牌的总供应量分配给所有者"。
    it("it: 部署应将令牌的总供应量分配给所有者", async function () {
        // 获取第一个账户（通常是部署者）。
        const [owner] = await ethers.getSigners();

        // 获取 Token 合约工厂。
        const Token = await ethers.getContractFactory("Token");

        // 部署 Token 合约。
        const hardhatToken = await Token.deploy();

        // 查询部署者账户的余额 -> 这个 balanceOf 函数是我们 contracts\Token.sol 中定义的。
        const ownerBalance = await hardhatToken.balanceOf(owner.address);

        // 使用 expect 断言检查部署后的总供应量是否等于部署者的余额。
        // 这个测试案例确保在部署合约后，总供应量的代币被分配给了部署者。
        expect(await hardhatToken.totalSupply()).to.equal(ownerBalance);
    });
});
```

我们目前不需要懂里面的一些逻辑 `ethers.getSigners`、 `Token.deploy`, 我们后续会学习他们, 我们现在只需要知道一些关键点:

- `describe` 用于组织测试用例，使测试结构更清晰。
    - 每个 describe 就是一次多个测试同时执行的分类
- `it` 用于定义具体的测试用例，每个测试用例都代表一个具体的行为验证。
    - 每个 it 就是一个需要执行的测试, 就好像我们请求一次 http 接口一样

## 解释测试代码

我们不需要很明白, 因为后续我们还会专门学习 `ethers.js ` 的一些用法。

现在我们逐行解释一下：

```js
// 获取第一个账户（通常是部署者）。
const [owner] = await ethers.getSigners();
```

**ethers.js** 中的 `Signer` 代表以太坊账户对象, 这里面包含了所有账户信息。

- 它用于将交易发送到合约和其他帐户。
- 在这里，我们获得了所连接节点中的帐户列表, 其实就是获取了这个区块链的所有账户
- 但是在本例中节点为**Hardhat Network**测试网络，仅保留第一个帐户, 也就是创建这个合约的账户
- 在实际的公共区块链上，你不能通过类似的方式获取所有账户的信息。

```js
// 获取 Token 合约工厂。
const Token = await ethers.getContractFactory("Token");
```

**ethers.js** 中的`ContractFactory`是用于部署新智能合约的抽象方法，因此此处的`Token`是用来实例代币合约的工厂。

```js
// 部署 Token 合约。
const hardhatToken = await Token.deploy();
```

在`ContractFactory`上调用`deploy()`将启动部署，并返回解析为`Contract`的`Promise`。

该 `hardhatToken` 对象包含了智能合约所有函数的方法, 可以通过 `hardhatToken` 对象调用我们定义的方法。

```js
// 查询部署者账户的余额 -> 这个 balanceOf 函数是我们 contracts\Token.sol 中定义的。
const ownerBalance = await hardhatToken.balanceOf(owner.address);
```

部署合约后，我们可以在`hardhatToken` 上调用合约方法，通过调用`balanceOf()`来获取所有者帐户的余额。

```js
// 使用 expect 断言检查部署后的总供应量是否等于部署者的余额。
// 这个测试案例确保在部署合约后，总供应量的代币被分配给了部署者。
expect(await hardhatToken.totalSupply()).to.equal(ownerBalance);
```

`totalSupply()` 返回代币的发行量，我们检查它是否等于`ownerBalance`部署者账户的余额。

**使用不同的账号测试**

如果你需要从默认帐户以外的其他帐户发送交易来测试代码，则可以在 ethers.js 的`Contract`中使用`connect()`方法来将其连接到其他帐户，像这样：

```js
// 使用 describe 定义一个测试套件，描述为 "Token 智能合约测试多个账户"。
describe("describe: Token 智能合约测试多个账户", function () {
    // 使用 it 定义一个测试案例，描述为 "应在账户之间转移代币"。
    it("it: 应在账户之间转移代币", async function () {
        // 获取三个账户：owner, addr1, addr2。
        const [owner, addr1, addr2] = await ethers.getSigners();

        // 获取 Token 合约工厂。
        const Token = await ethers.getContractFactory("Token");

        // 部署 Token 合约。
        const hardhatToken = await Token.deploy();

        // 从 owner 部署账户转移 50 个代币到 addr1。
        await hardhatToken.transfer(addr1.address, 50);

        // 使用 expect 断言检查 addr1 的余额是否为 50。
        expect(await hardhatToken.balanceOf(addr1.address)).to.equal(50);

        // 使用 connect 方法切换到 addr1 的上下文，以便 addr1 能够发起交易。并 从 addr1 转移 50 个代币到 addr2。
        await hardhatToken.connect(addr1).transfer(addr2.address, 50);

        // 使用 expect 断言检查 addr2 的余额是否为 50。
        expect(await hardhatToken.balanceOf(addr2.address)).to.equal(50);
    });
});
```
