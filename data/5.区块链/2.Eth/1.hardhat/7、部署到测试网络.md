## 部署到测试网络

部署合约有2种方式:

1. Ignition点火: 单独为复杂部署独立出来的部署方式
2. script脚本: 简单的部署通过脚本即可

**一般来说, 项目都不会很大, 我们使用 script 脚本部署即可。**

## 1. Ignition点火

hardhat 框架给我们提供了简单的部署方案。

我在 `ignition\modules` 目录中新增 deploy.js (默认生成的 Lock.js 已经删除)

```js
// 导入 buildModule 函数，这是 Hardhat Ignition 模块化部署框架的一部分。
const { buildModule } = require("@nomicfoundation/hardhat-ignition/modules");
// 需要部署的合约名称, 如果我们需要换合约部署, 只需要更改这里的名称即可
const contractName = "Token"

// 使用 buildModule 构建一个模块化的部署脚本。
module.exports = buildModule("deploy", (m) => {
  // 创建合约实例 -> (合约名称, 构造器参数, 部署选项)
  const contract = m.contract(contractName, [], {});

  // 返回合约实例。
  return { contract };
});
```

运行到本地网络:


# 2. script脚本

创建一个 `scripts` 目录, 并创建 `deploy.js` 脚本文件:

```js

const { ethers } = require("hardhat")

// 需要部署的合约名称, 如果我们需要换合约部署, 只需要更改这里的名称即可
const contractName = "Token"

// 定义一个异步函数 main，用于部署 Token 合约。
async function main() {
    const [deployer] = await ethers.getSigners();
    console.log("部署合约帐户地址:", deployer.address);

    // 获取账户的余额
    const balance = await deployer.provider.getBalance(deployer.address);
    // https://eth-converter.com/
    console.log("账户余额 balance(Wei):", balance.toString());
    // 将余额转换为以太币 (ETH)
    const balanceInEther = ethers.formatEther(balance);
    // 输出余额（以 ETH 为单位）
    console.log("账户余额 balance(ETH):", balanceInEther);

    console.log("___________________________________________________________")
    console.log("部署合约...")
    // 获取合约工厂。
    const contractFactory = await ethers.getContractFactory(contractName);
    // 部署合约
    const contract = await contractFactory.deploy();
    console.log(`合约地址: ${contract.target}`);
}

// 执行 main 函数，并处理可能发生的错误。
main()
    .then(() => process.exit(0)) // 如果部署成功，则退出进程。
    .catch(error => {
        console.error(error); // 如果发生错误，则输出错误信息。
        process.exit(1); // 退出进程，并返回错误代码 1。
    });
```

运行到本地网络:
